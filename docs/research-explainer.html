Music Psychology · 1936

Hevner's Adjective Circle

The first systematic map of how music feels — and the foundation of this project's chord corpus.

In 1936, Kate Hevner played excerpts of classical music to listeners and asked them to circle adjectives from a checklist that matched the mood. She then grouped adjectives that were frequently circled together and arranged the groups in a circle so that emotionally adjacent clusters sat next to each other.

The result: eight clusters, arranged so that moving clockwise takes you from dignified → sad → dreamy → serene → graceful → happy → exciting → vigorous and back again. Neighboring clusters share emotional territory; opposite clusters are maximally different.

Why a Circle?

The circular arrangement isn't decorative — it encodes a continuous emotional topology. There are no hard boundaries between clusters; the space is smooth. "Sad" and "dreamy" bleed into each other; "vigorous" and "exciting" overlap. This is why embedding-based classification (which also produces a smooth similarity space) is such a natural fit for mood-based chord selection.

The Gap Between 1936 and This Project

Hevner described the emotional character of full musical pieces, not individual chords. Applying her clusters to single chords required a further inference step: empirical chord-emotion research (from the 1990s onward) showed that major chords cluster in the upper-right of the circle (happy, graceful) while minor chords fall in the upper-left and lower regions (dignified, sad, dreamy). The dominant chord (V) sits in the exciting/vigorous zone; the tonic (I) occupies the serene/graceful region when approached from dominant.

In this project

Each chord's keyword page in data/chords/*.md is a hand-curated Hevner-style adjective list. The "Core Adjectives" section maps directly to one or two Hevner clusters. The embedding model then turns those adjective pages into vectors — essentially doing computationally what Hevner's listeners did perceptually: placing emotional descriptions in a continuous space where similar feelings live close together.

Later Refinements

Farnsworth (1954) extended the circle to 10 groups. Other researchers merged it with Russell's two-dimensional valence-arousal model. But Hevner's original structure remains the most widely cited categorical framework in music emotion research — it still appears in papers published in 2024.

"The structure of the adjective circle corresponds to the structure of musical affect — both are smooth, continuous, and without natural category boundaries."
— Paraphrase of Hevner (1936)
Zero-Shot Classification · 2022

Lbl2Vec

Classify documents using only natural-language descriptions of categories — no labeled training examples needed.

Lbl2Vec (Schopf, 2022) is a zero-shot text classification technique. The core insight: if you can describe what a category means in plain language, you can classify new documents by measuring how similar they are to those descriptions — in embedding space.

No labeled examples. No fine-tuning. Just: describe your categories → embed them → embed the query → pick the closest.

The Algorithm

Click each step to expand it.

1
Write label descriptions
For each category, write a set of representative keywords or phrases.
You don't need labeled data — you need knowledge about what each category means. This is expressed as a short keyword list or paragraph per category.

In the original Lbl2Vec paper, label descriptions were a handful of keywords (e.g. for a "sports" category: "athlete, game, score, team, tournament"). This project uses much richer pages — hundreds of adjectives and phrases per chord — which produces stronger, more distinctive anchor vectors.
2
Embed the descriptions → label vectors
Run each description through an embedding model to get a fixed-size vector.
Each category description becomes a single vector in embedding space. These are called prototypes or anchors. They are computed once and cached — this is the "build time" step.

In this project: python src/anchor_builder.py reads all 7 chord pages, encodes each one, and saves the result to data/anchors.pt — a (7 × 1024) tensor.
3
Embed the query document
At classification time, embed the input text with the same model.
The query is encoded using the same embedding model and the same instruction prefix used at build time. This is critical — if the query and label vectors were encoded differently, the similarity scores would be meaningless.

In this project: the instruction prefix "Instruct: Represent this text by its emotional and affective character…" is applied to both chord pages at build time and to the input text at query time.
4
Cosine similarity → scores
Compute the cosine similarity between the query vector and every label vector.
Cosine similarity measures the angle between two vectors — 1.0 means identical direction, 0 means orthogonal, −1 means opposite. Because both vectors are L2-normalized, this is equivalent to a dot product.

Result: one score per category. Higher score = the query is more like that label's description.
5
Pick the winning label
Return the category with the highest similarity, optionally with a confidence threshold.
Original Lbl2Vec uses argmax. This project uses temperature sampling: compute softmax(scores / T) at T=0.05, then sample. This is nearly identical to argmax for clear-cut cases (large gap between best and second-best) but produces natural variety for ambiguous inputs rather than always collapsing to the same default chord.

If the best score is below 0.40 (the confidence threshold), the result is flagged low_confidence=True.

What Makes This Project's Version Richer

Standard Lbl2Vec uses a short keyword list (5–10 words) per label. This project uses full keyword pages — 20–30 adjectives plus 10–15 extended phrases per chord, sourced from Hevner's research. Richer descriptions produce more distinctive anchor vectors, which improves separation between neighboring emotional categories (e.g. "longing" vs. "sadness").

The anchor quality check: after building, anchor_builder.py prints a 7×7 pairwise cosine similarity matrix. All off-diagonal values should be below 0.85. If any pair is above that threshold, those two chord pages are too emotionally similar and need to be revised.
In this project

The name "anchor" comes from this family of techniques — precomputed label prototypes that "anchor" each category in embedding space. The instruction-aware Qwen3-Embedding model gives this project an edge over standard Lbl2Vec: by telling the model to focus on emotional and affective character, the anchor vectors are tuned to the right dimension of meaning rather than general semantic content.

Dataset · ISMIR 2021

EMOPIA

The only public dataset that links chord progressions to emotion labels — a potential ground truth for this project's chord→emotion mappings.

EMOPIA (Hung et al., ISMIR 2021) is a dataset of 1,087 pop piano clips from 387 songs, each annotated with an emotion quadrant. It was the first dataset to combine both chord information and listener emotion labels in the same resource.

The Valence-Arousal Model

EMOPIA uses Russell's Circumplex Model of Affect (1980) as its emotion framework. Rather than named categories (like Hevner's clusters), Russell proposed two continuous axes:

  • Valence — how positive or negative the feeling is (left = negative, right = positive)
  • Arousal — how activated or calm the feeling is (down = calm, up = excited)

Every emotion can be placed somewhere on this 2D plane. EMOPIA simplifies this to four quadrants (Q1–Q4). Click a quadrant to see examples.

Q1 — High Valence, High Arousal
Happy, exciting, joyful, energetic, triumphant
Q2 — Low Valence, High Arousal
Angry, tense, agitated, fearful, anxious
Q3 — Low Valence, Low Arousal
Sad, melancholic, gloomy, depressed, mournful
Q4 — High Valence, Low Arousal
Peaceful, serene, calm, tender, content

What EMOPIA Contains

1,087 clips · 387 songs · 4 emotion quadrants · MIDI available · chord progressions extractable

Each clip is ~30 seconds of solo piano. The emotion label was assigned by the dataset creators based on the musical content (tempo, dynamics, mode, chord progression).

EMOPIA+ (2024, Zenodo 13122742) extended this by extracting lead sheets — melody plus chord progressions — from 367 of the clips, with manual key signature corrections. This is the version that makes the chord→emotion connection directly inspectable.

Why It Matters for This Project

This project's chord keyword pages were written by hand, guided by Hevner's research and musical intuition. EMOPIA+ provides an empirical check: you could aggregate all chord occurrences in Q3 clips (sad/melancholic) and see if vi and ii dominate. If they do, the keyword page assignments are validated. If they don't, it's a signal that a page might need revision.

Potential use in this project

Run the EMOPIA+ chord data through the classifier and compare the predicted quadrant against the annotated quadrant. A high agreement rate would mean the anchor embeddings have successfully internalized the same emotion-to-chord mappings that human listeners perceive in recorded music. A low agreement rate would identify which chord pages need work.
